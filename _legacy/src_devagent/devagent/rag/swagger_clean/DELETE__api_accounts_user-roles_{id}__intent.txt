TYPE: INTENT

SUMMARY:
Remover vínculo de role do usuário

DESCRIPTION:
Remove um vínculo `UserRole` existente.

**Autenticação:** requer sessão ativa via cookie HttpOnly `gpp_session_<APP>` (ex: `gpp_session_PORTAL`). O frontend deve enviar todas as requisições com `withCredentials: true`.

**Permissões:** acesso exclusivo a usuários com perfil `PORTAL_ADMIN`.

**Efeito colateral:** após a remoção do vínculo, o endpoint executa `sync_user_permissions(user)` para recalcular as permissões efetivas do usuário com base nas roles remanescentes.
