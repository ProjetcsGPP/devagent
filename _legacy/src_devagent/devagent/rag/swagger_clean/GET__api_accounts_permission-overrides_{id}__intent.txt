TYPE: INTENT

SUMMARY:
Detalhar override de permissão

DESCRIPTION:
Retorna os dados de um override de permissão específico.

**Autenticação:** requer sessão ativa via cookie HttpOnly `gpp_session_<APP>` (ex: `gpp_session_PORTAL`). O frontend deve enviar todas as requisições com `withCredentials: true`.

**Permissões:** acesso exclusivo a usuários com perfil `PORTAL_ADMIN`.

**Impacto funcional:** overrides afetam as permissões efetivas do usuário. As mutações neste recurso disparam `sync_user_permissions(user)` para recalcular imediatamente o estado efetivo.
