TYPE: INTENT

SUMMARY:
Vincular role a usuário

DESCRIPTION:
Cria um vínculo `UserRole` entre usuário, role e aplicação.

**Autenticação:** requer sessão ativa via cookie HttpOnly `gpp_session_<APP>` (ex: `gpp_session_PORTAL`). O frontend deve enviar todas as requisições com `withCredentials: true`.

**Permissões:** acesso exclusivo a usuários com perfil `PORTAL_ADMIN`.

**Efeito colateral:** após a criação do vínculo, o endpoint executa `sync_user_permissions(user)` para sincronizar as permissões efetivas do usuário de forma atômica.
