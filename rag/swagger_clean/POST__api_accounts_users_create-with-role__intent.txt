TYPE: INTENT

SUMMARY:
Criar usuário com role (fluxo completo)

DESCRIPTION:
Cria atomicamente `auth.User` + `UserProfile` + `UserRole` e dispara sincronização de permissões via `sync_user_permissions`.

**Autenticação:** requer sessão ativa via cookie HttpOnly `gpp_session_<APP>` (ex: `gpp_session_PORTAL`). O frontend deve enviar todas as requisições com `withCredentials: true`.

**Permissões:**
- `401 Unauthorized` — sessão ausente ou expirada (não autenticado).
- `403 Forbidden (papel insuficiente)` — autenticado, mas não é `PORTAL_ADMIN` nem `superuser`.
- `403 Forbidden (escopo)` — é `PORTAL_ADMIN`, mas a `aplicacao` de destino está fora das aplicações que o operador gerencia.

**Atomicidade:** `User`, `UserProfile`, `UserRole` e o sync de permissões ocorrem em uma única transação — em caso de falha, nenhum registro parcial é persistido.

**Resposta:** retorna um `dict` customizado com os dados do usuário criado, sua role e a aplicação vinculada.
